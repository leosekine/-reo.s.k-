# Attendance App

## 目的

小規模チーム（10〜30人程度）の勤怠管理を効率化するWebアプリ。
紙やスプレッドシートで勤怠を管理しているチームが、打刻漏れ・シフト調整の手間・日報の散逸に困っている。
このアプリで打刻・シフト・日報を一元管理し、管理者の確認作業を削減する。

## 画面構成（5画面）

1. **打刻画面** - 出勤・退勤・休憩のワンタップ打刻
2. **シフト申請画面** - シフト希望の提出・編集
3. **カレンダー画面** - 月間の勤務状況を一覧表示
4. **日報画面** - その日の業務内容を記録・提出
5. **ダッシュボード** - 管理者向けのチーム全体の勤怠サマリー

## テーブル設計（5テーブル）

| テーブル名 | 主な用途 |
|---|---|
| profiles | ユーザー情報（名前・役割・所属） |
| attendances | 打刻記録（出勤・退勤・休憩の時刻） |
| shifts | シフト申請（希望日時・ステータス） |
| daily_reports | 日報（業務内容・所感） |
| action_logs | 操作履歴（誰がいつ何をしたか） |

## 承認フロー

シフト申請と日報は以下のステータスで管理する:

```
申請中 (pending) → 承認 (approved)
                  → 却下 (rejected)
```

- メンバーが申請を提出すると `pending` になる
- 管理者が確認して `approved` または `rejected` に変更する
- 却下時はコメントを付けて差し戻せる

## 役割（ロール）

| 役割 | できること |
|---|---|
| **admin**（管理者） | 全メンバーの打刻・シフト・日報の閲覧、承認・却下、ダッシュボード利用 |
| **member**（一般） | 自分の打刻・シフト申請・日報提出、自分のカレンダー閲覧 |
